# 11. `rg`: быстрый поиск по всей папке

После `grep` быстро становится заметно: искать по всей папке можно, но команды выходят шумными. `rg` решает ту же задачу, но обычно короче и удобнее, когда работа идёт по целому набору файлов.

Перейди в папку:

```bash
cd game_server_intro
```

## Базовый поиск

```bash
rg 'AuthFailed'
```

Должно быть: находятся совпадения только в файлах из `logs`.

`rg` ищет по папке рекурсивно по умолчанию. Это главное отличие от базового `grep`.

## Номера строк

```bash
rg -n 'ERROR' logs
```

Должно быть: в выводе есть имя файла, номер строки и сама строка.

## Только имена файлов

```bash
rg -l 'rate_limited' logs
```

Должно быть:
`logs/2026-02-20.log`
`logs/2026-02-21.log`
`logs/2026-02-22.log`

## Подсчёт совпадений

```bash
rg -c 'PlayerConnected' logs
```

Должно быть:
`logs/2026-02-20.log:5`
`logs/2026-02-21.log:4`
`logs/2026-02-22.log:6`

## Фильтр по шаблону файла

```bash
rg -n -g '*.log' 'Skin_Dragon'
```

Должно быть: совпадения находятся только в `.log` файлах.

Например:
`logs/2026-02-22.log:5:...`

## Шпаргалка

- `rg 'text'` искать по всей папке.
- `rg -n 'text' path` показать номера строк.
- `rg -l 'text' path` показать только имена файлов.
- `rg -c 'text' path` посчитать совпадения по файлам.
- `rg -g '*.log' 'text'` искать только в нужных файлах.

## Практика

1. Найди все строки с `AuthFailed` по всей папке.

Должно быть: находятся совпадения только в логах.

2. Покажи только имена файлов, где встречается `rate_limited`.

Должно быть:
`logs/2026-02-20.log`
`logs/2026-02-21.log`
`logs/2026-02-22.log`

3. Посчитай, сколько раз `PlayerConnected` встречается в каждом дневном логе.

Должно быть:
`logs/2026-02-20.log:5`
`logs/2026-02-21.log:4`
`logs/2026-02-22.log:6`

4. Найди `Skin_Dragon`, но ищи только в `.log` файлах.

Должно быть: в выводе есть только строки из `logs/*.log`.
