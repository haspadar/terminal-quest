# 04. `sort`: наводим порядок

После `find` и `wc` быстро появляется новая проблема: нужные строки уже есть, но читать их неудобно. Где-то хочется алфавитный список, где-то обратный порядок, а где-то числа надо расставить по возрастанию. Для этого и нужен `sort`.

`sort` очень часто живёт в пайпе. Одна команда что-то находит или считает, а `sort` потом раскладывает строки в понятном порядке.

Перейди в папку:

```bash
cd game_server_intro
```

## Обычная сортировка

По умолчанию `sort` сортирует строки как текст. Если на входе уже есть список путей или имён, этого часто достаточно.

```bash
find matches -name 'match_*.txt' | sort
```

Должны выстроиться пути к матчам от `match_1001.txt` до `match_1004.txt`:

- `matches/match_1001.txt`
- `matches/match_1002.txt`
- `matches/match_1003.txt`
- `matches/match_1004.txt`

Здесь `find` находит файлы, а `sort` просто раскладывает их по алфавиту.

## Обратный порядок: `-r`

Когда нужен тот же список, но с конца, добавь `-r`.

```bash
find matches -name 'match_*.txt' | sort -r
```

Должны выстроиться те же файлы, но в обратном порядке:

- `matches/match_1004.txt`
- `matches/match_1003.txt`
- `matches/match_1002.txt`
- `matches/match_1001.txt`

Флаг `-r` не меняет сам принцип сортировки. Он просто переворачивает готовый порядок.

## Числовая сортировка: `-n`

По умолчанию `sort` сравнивает строки как текст. Для чисел это неудобно: строка `10` для него не число, а просто набор символов. Поэтому для числовых столбцов нужен `-n`.

Здесь удобно взять уже знакомый `wc -c`: он показывает размер файла в байтах в первом столбце.

```bash
wc -c chat/*.log | sort -n
```

Сначала должен появиться меньший лог, потом больший, а потом строка с суммой:

- `124 chat/clan_dragons.log`
- `279 chat/global_chat.log`
- `403 total`

Здесь как раз видно, зачем нужен `-n`: сортировка идёт по числам в начале строки, а не по тексту целиком.

## Сортировка по колонке: `-k`

Иногда число или имя лежит не в начале строки, а дальше. Тогда `sort` нужно подсказать, по какой колонке смотреть.

В выводе `ls -l` размер файла стоит в пятой колонке. Значит, можно отсортировать список инвентарей именно по размеру.

```bash
ls -l items/*.csv | sort -k5,5n
```

Наверху должны оказаться самые маленькие файлы, а внизу самый большой:

- `items/li_inventory.csv`
- `items/max_inventory.csv`
- `items/john_inventory.csv`
- `items/anna_inventory.csv`
- `items/carlos_inventory.csv`
- `items/kenji_inventory.csv`
- `items/ivan_inventory.csv`

Запись `-k5,5n` читается так:

- `k5,5` — сортировать по пятой колонке;
- `n` — считать эту колонку числовой.

Если нужно, флаги можно сочетать. Например, `-k5,5nr` отсортирует ту же колонку по числам, но в обратном порядке.

## Убрать повторы: `-u`

Иногда после сортировки хочется сразу оставить только уникальные строки. Для этого у `sort` есть флаг `-u`.

Здесь удобно взять список папок первого уровня: в нём одинаковых строк нет, но на искусственном примере легко увидеть, как это работает.

```bash
printf '%s\n' logs logs chat chat items items | sort -u
```

Должны остаться только уникальные строки:

- `chat`
- `items`
- `logs`

Флаг `-u` одновременно сортирует строки и убирает повторы.

## Шпаргалка

- `sort` отсортировать строки как текст.
- `sort -r` перевернуть порядок.
- `sort -n` сортировать строки как числа.
- `sort -k5,5n` сортировать по пятой колонке как по числу.
- `sort -u` отсортировать и убрать повторы.

## Напоследок

1. GNU `sort` очень хорошо оптимизирована.
Она умеет сортировать данные, которые не помещаются в память, использовать временные файлы на диске и распараллеливать работу на больших наборах.

2. Сортировка когда-то была настолько важной задачей, что для неё делали отдельные аппаратные устройства.
В 70-х и 80-х обработка больших текстовых файлов была серьёзной нагрузкой, и `sort` был одним из ключевых инструментов в этой работе.
