# 19. `fzf`: интерактивный поиск прямо в терминале

Когда файлов много, а имя нужного помнится лишь частично — `ls` и `fd` дают список, но выбирать из него приходится глазами. `fzf` решает иначе: показывает список и фильтрует его по мере набора, прямо в терминале.

Перейдите в папку:

```bash
cd game_server_intro
```

## Установка

Ubuntu / WSL:

```bash
sudo apt install fzf
```

Arch:

```bash
sudo pacman -S fzf
```

После установки проверьте:

```bash
fzf --version
```

Если команда `fzf --fish` дальше в уроке не работает, причина обычно в старой версии из `apt`. На Ubuntu и в WSL такое бывает: ставится версия около `0.41`, в которой генерация привязок для `fish` ещё недоступна. На Arch пакет обычно свежий и этот шаг не нужен.

В этом случае удалите пакет из `apt` и поставьте свежую версию отдельно:

```bash
sudo apt remove fzf
git clone --depth 1 https://github.com/junegunn/fzf.git ~/.fzf
~/.fzf/install
```

В установщике включите интеграцию для shell и key-bindings. После этого снова проверьте:

```bash
fzf --version
fzf --fish
```

Версия должна быть свежей, а `fzf --fish` должен печатать код для `fish`, а не ошибку.

## Базовый запуск

```bash
fzf
```

`fzf` показывает список всех файлов в текущей папке рекурсивно. Начните набирать — список сужается в реальном времени. Стрелки вверх/вниз перемещают курсор, Enter выбирает файл и печатает его путь. Нажмите Esc или Ctrl+C, чтобы выйти без выбора.

Наберите `log` — должны остаться только файлы, в пути которых есть эти буквы.

Fuzzy-поиск не требует точного совпадения: `lgcln` найдёт `logs/clan_dragons.log`.

## Фильтрация произвольного списка через pipe

`fzf` читает строки из stdin — через pipe можно фильтровать что угодно.

Передайте список файлов матчей:

```bash
ls matches/ | fzf
```

Список из четырёх файлов появится в интерфейсе `fzf`. Наберите `1003` — должен остаться `match_1003.txt`.

Можно фильтровать содержимое файла:

```bash
cat logs/2026-02-22.log | fzf
```

Наберите `ERROR` — должны остаться только строки с ошибками.

## Подстановка `(fzf)`: выбрать и сразу открыть

Самый частый способ использовать `fzf` — вставить выбранный путь прямо в команду:

```fish
cat (fzf)
```

`fzf` откроется в интерактивном режиме. Найдите `server.ini` — наберите `serv` — и нажмите Enter. Содержимое файла сразу появится на экране.

То же самое работает с любой командой:

```fish
rg 'ERROR' (fzf)
```

Найдите любой `.log` файл — `rg` выполнится именно для него.

## История команд и Ctrl+R

Терминал запоминает все команды, которые вы запускали — это работает в любом шелле: bash, zsh, fish. Нажмите **Ctrl+R** прямо сейчас — появится список истории. Стрелками можно листать, Enter — выполнить команду снова.

Попробуйте: запустите `fzf --version`, потом нажмите Ctrl+R и наберите `fzf` — нужная строка найдётся сразу.

Это работает и без `fzf`, но список неудобный: он показывается как простой текст и нельзя нормально фильтровать. `fzf` это исправит.

## Горячие клавиши в fish

Если `fzf --fish` отрабатывает без ошибки, добавьте привязки в конфиг:

```fish
echo 'fzf --fish | source' >> ~/.config/fish/config.fish
source ~/.config/fish/config.fish
```

После этого Ctrl+R откроет интерактивный поиск по истории через `fzf` — с фильтрацией по мере набора. Ещё заработает:

- **Ctrl+T** — вставить путь к файлу прямо в текущую команду. Например, наберите `cat`, нажмите Ctrl+T, выберите файл — путь вставится автоматически.

## Связка с `fd` и предпросмотр

По умолчанию `fzf` использует встроенный обход файловой системы. Его можно заменить на `fd` — он быстрее и уважает `.gitignore`. Задайте переменную окружения:

```fish
set -x FZF_DEFAULT_COMMAND 'fd --type f'
```

Теперь просто `fzf` будет использовать `fd` для построения списка.

Добавьте предпросмотр содержимого прямо в интерфейсе:

```fish
fzf --preview 'cat {}'
```

Правая панель показывает содержимое файла, на котором стоит курсор. Перейдите стрелками по файлам `matches/` — правая часть будет обновляться без Enter.

Чтобы не набирать `--preview` каждый раз, добавьте в конфиг:

```fish
set -x FZF_DEFAULT_OPTS '--preview "cat {}"'
```

## Шпаргалка

- `fzf` — интерактивный выбор файла из текущей папки.
- `cmd | fzf` — фильтровать произвольный список строк.
- `cat (fzf)` — выбрать файл и передать его команде.
- `fzf --preview 'cat {}'` — показывать содержимое при навигации.
- `set -x FZF_DEFAULT_COMMAND 'fd --type f'` — использовать `fd` как источник списка.
- Ctrl+R — поиск по истории fish (после подключения key-bindings).
- Ctrl+T — вставить путь к файлу в текущую команду.

## Напоследок

- `fzf` написан на Go и работает даже на списках из миллиона строк без заметной задержки.
- Fuzzy-алгоритм `fzf` ставит наверх совпадения, где буквы идут подряд — поэтому `er22` найдёт `2026-02-22.log` точнее, чем `log22`.

## Практика

19.1. Запустите `cat logs/2026-02-22.log | fzf` и найдите строки с `Purchase`.

   Должны остаться пять строк о покупках — `RaidTicket`, `Skin_Dragon` и другие.

19.2. С помощью подстановки `(fzf)` откройте любой профиль игрока из `players/`.

   Наберите имя игрока в `fzf`, выберите его файл — содержимое профиля должно появиться в терминале.

19.3. Задайте `set -x FZF_DEFAULT_COMMAND 'fd --type f'` и запустите `fzf --preview 'cat {}'`. Найдите файл `server.ini`.

   В левой панели должен появиться `configs/server.ini`, в правой — его содержимое с секцией `[server]` и настройками сервера.
