# 21. `jq`: работаем с JSON в терминале

В прошлом уроке вы видели, что `rg` может найти строку в JSON-файле, но не умеет работать с его структурой. `jq` — это инструмент, который понимает JSON: умеет доставать поля, сортировать, фильтровать и группировать данные.

## Установка

Ubuntu / WSL:

```bash
sudo apt install jq
```

Arch:

```bash
sudo pacman -S jq
```

## Данные для урока

В папке `albion/prices/` лежат реальные цены из игры Albion Online — такие данные возвращает публичный API. Данные собраны за три дня и разложены по папкам:

```
albion/prices/
├── 2026-03-24/
│   ├── T6_BAG.json
│   ├── T6_METALBAR.json
│   └── T6_PLANKS.json
├── 2026-03-25/
│   └── ...
└── 2026-03-26/
    └── ...
```

Каждый файл — массив объектов: цена продажи и покупки по каждому городу за один день.

Перейдите в папку:

```fish
cd albion/prices
```

## Онлайн-песочница

Прежде чем разбирать команды — откройте **https://play.jqlang.org/** в браузере. Вставьте содержимое любого файла в левую панель, в строку фильтра пишите команды из урока. Результат появится справа сразу.

## Как читать фильтры jq

В `jq` фильтр — это маршрут по структуре JSON. Три строительных блока:

- **`.`** — «я стою здесь». Сама по себе — весь документ. После поля — «зайди внутрь»: `.city` значит «возьми значение ключа `city`».
- **`[]`** — «пройди по всем элементам». Если стоишь на массиве, `[]` выдаёт каждый элемент по очереди.
- **`|`** — передать результат дальше, как pipe в терминале.

Разберём на примере. Файл — это массив объектов. Чтобы достать город из каждого:

```
.          →  весь файл (массив)
.[]        →  каждый объект из массива
.[].city   →  поле city из каждого объекта
```

## Красивый вывод

```fish
jq '.' 2026-03-26/T6_BAG.json
```

Файл с отступами и цветами. Видно: массив из семи объектов, каждый — цена в одном городе.

## Поле каждого объекта

```fish
jq '.[].city' 2026-03-26/T6_BAG.json
```

`.[]` — пройти по всем элементам, `.city` — взять поле `city`. Должны появиться семь городов.

Без кавычек — флаг `-r`:

```fish
jq -r '.[].city' 2026-03-26/T6_BAG.json
```

## Сортировка: sort_by

`sort_by` сортирует массив по значению поля. Покажите города от дешёвых к дорогим:

```fish
jq '[.[] | {city, price: .sell_price_min}] | sort_by(.price)' 2026-03-26/T6_BAG.json
```

Здесь `{city, price: .sell_price_min}` создаёт новый объект только с нужными полями. Должны появиться города в порядке роста цены — Fort Sterling и Thetford внизу, Bridgewatch наверху.

## Минимум и максимум: min_by, max_by

Где сегодня дешевле всего купить T6_BAG:

```fish
jq 'min_by(.sell_price_min) | {city, price: .sell_price_min}' 2026-03-26/T6_BAG.json
```

Должен появиться один объект — город с минимальной ценой.

Где дороже всего продать:

```fish
jq 'max_by(.sell_price_min) | {city, price: .sell_price_min}' 2026-03-26/T6_BAG.json
```

## Арбитраж: найти разницу

Купить в самом дешёвом городе и продать в самом дорогом — базовая гипотеза арбитража. Вот как это выглядит через `jq`:

```fish
jq '
  (min_by(.sell_price_min) | {buy_city: .city, buy_price: .sell_price_min}) +
  (max_by(.sell_price_min) | {sell_city: .city, sell_price: .sell_price_min})
' 2026-03-26/T6_BAG.json
```

Должно появиться что-то вроде:

```json
{
  "buy_city": "Thetford",
  "buy_price": 42000,
  "sell_city": "Bridgewatch",
  "sell_price": 59956
}
```

Разница ~18000 серебра на одном предмете — стоит ли везти, зависит от стоимости перемещения.

## Фильтрация: select

`select()` оставляет только элементы, для которых условие истинно. Найдите города, где `sell_price_min` меньше 50000:

```fish
jq -r '[.[] | select(.sell_price_min < 50000)] | sort_by(.sell_price_min) | .[] | "\(.city): \(.sell_price_min)"' 2026-03-26/T6_BAG.json
```

`"\(.city): \(.sell_price_min)"` — строка с подстановкой значений. Должны появиться города с ценой ниже 50000, отсортированные по возрастанию.

## Шпаргалка

- `jq '.' file.json` — красивый вывод.
- `jq '.[].field'` — поле каждого элемента.
- `jq 'sort_by(.field)'` — сортировка массива.
- `jq 'min_by(.field)'` — элемент с минимальным значением.
- `jq 'max_by(.field)'` — элемент с максимальным значением.
- `jq '.[] | select(.field < N)'` — фильтрация по условию.
- `jq '{key: .field}'` — новый объект с нужными полями.
- `jq -r '"\(.field1): \(.field2)"'` — строка с подстановкой.

## Напоследок

- `jq` создал Стивен Дентон в 2012 году, вдохновившись идеей `sed` и `awk`, но для JSON. Сегодня это стандартный инструмент в DevOps и при работе с API.
- Файлы в `albion/prices/` — реальные данные с публичного API Albion Online Data Project. Позже можно запрашивать их прямо в терминале через `curl` и сразу обрабатывать через `jq`.
