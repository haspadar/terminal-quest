# 09. `cut`: вырезаем нужные поля

После `find`, переменных и `printf` часто остаётся ещё одна задача: строка уже есть, но из неё нужен только кусок. Например, значение после `=` или часть пути.

Для этого и нужен `cut`.

Перейдите в папку:

```bash
cd game_server_intro
```

Дальше все команды запускайте внутри неё.

## Базовая форма

У `cut` три главные части:

```bash
cut -dDELIMITER -fFIELDS FILE
```

- `-d` задаёт разделитель;
- `-f` говорит, какие поля оставить.

Например:

```bash
cut -d= -f2 players/europe/anna.txt
```

Должны быть видны:

- `anna`
- `europe`
- `38`
- `healer`
- `8200`
- `dragons`
- `anna@test.org`
- `2026-02-19`

Здесь `cut` берёт каждую строку вида `key=value`, делит её по `=` и оставляет второе поле.

Наберите:

```bash
cut -d= -f1 players/europe/anna.txt
```

Должны быть видны:

- `nickname`
- `region`
- `level`
- `class`
- `gold`
- `clan`
- `email`
- `created_at`

## Несколько полей сразу

`cut` умеет работать не только с содержимым файла, но и с любыми строками, где есть повторяющийся разделитель.

```bash
printf 'players/europe/anna.txt\n' | cut -d/ -f2,3
```

Должна остаться строка `europe/anna.txt`.

Здесь разделитель — `/`, а `cut` оставляет второе и третье поля.

Наберите:

```bash
find players -name '*.txt' | sort | cut -d/ -f2,3
```

Должны быть видны:

- `america/carlos.txt`
- `america/john.txt`
- `asia/kenji.txt`
- `asia/li.txt`
- `europe/anna.txt`
- `europe/ivan.txt`
- `europe/max.txt`

## Диапазон полей

Если нужны не отдельные поля, а всё начиная с какого-то места, удобнее использовать диапазон.

```bash
printf 'players/europe/anna.txt\n' | cut -d/ -f2-
```

Должна остаться строка `europe/anna.txt`.

Запись `-f2-` значит: взять второе поле и всё, что идёт дальше.

Наберите:

```bash
find players -name '*.txt' | sort | cut -d/ -f2-
```

Должны быть видны:

- `america/carlos.txt`
- `america/john.txt`
- `asia/kenji.txt`
- `asia/li.txt`
- `europe/anna.txt`
- `europe/ivan.txt`
- `europe/max.txt`

## `cut` может читать из пайпа

`cut` умеет работать не только с файлами, но и со `stdin`. Это хорошо сочетается с `printf`.

```bash
printf 'players/europe/anna.txt\n' | cut -d/ -f3
```

Должна остаться строка `anna.txt`.

Так удобно быстро проверить шаблон до того, как применять его к настоящим файлам.

Наберите:

```bash
printf 'level=38\n' | cut -d= -f2
```

Должна остаться строка `38`.

## Когда `cut` особенно полезен

`cut` хорошо работает там, где строки уже устроены просто и ровно:

- `key=value`;
- пути вроде `players/europe/anna.txt`;
- повторяющиеся поля с одним разделителем.

Если структура у строки простая, `cut` обычно короче и быстрее, чем более тяжёлые инструменты.

## Шпаргалка

- `cut -d= -f2 file` взять значение после `=`.
- `cut -d/ -f2,3 file` взять несколько полей.
- `cut -d/ -f2- file` взять поле и всё, что идёт дальше.
- `... | cut -d= -f2` применять `cut` к выводу другой команды.

## Напоследок

1. `cut` особенно удобен там, где данные уже лежат в ровном виде: одинаковый разделитель и одинаковый порядок полей в каждой строке.

2. Очень часто `cut` работает не с файлами напрямую, а с выводом другой команды. Поэтому его постоянно ставят в середину пайпа как короткий фильтр.
