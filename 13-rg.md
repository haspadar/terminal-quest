# 13. `rg`: быстрый поиск по всей папке

После `grep` быстро становится заметно: искать по всей папке можно, но команда выходит шумной. `rg` решает ту же задачу короче и удобнее, когда работа идёт сразу по набору файлов.

Перейдите в папку:

```bash
cd game_server_intro
```

Дальше все команды запускайте внутри неё.

## Базовая форма

У `rg` самая частая форма такая:

```bash
rg 'PATTERN' PATH
```

- `PATTERN` — что искать;
- `PATH` — где искать.

Например:

```bash
rg 'AuthFailed' .
```

Должны найтись совпадения только в дневных логах.

Главное отличие от базового `grep`: `rg` сам обходит папки рекурсивно.

Наберите:

```bash
rg 'PlayerConnected' logs
```

Должны быть видны строки с подключением игроков из трёх файлов `logs/*.log`.

## Номера строк: `-n`

Когда совпадений много, полезно сразу видеть номера строк.

```bash
rg -n 'ERROR' logs
```

Должны быть видны имя файла, номер строки и сама строка.

Наберите:

```bash
rg -n 'region=europe' players
```

Должны быть видны три строки из европейских профилей.

## Только имена файлов: `-l`

Если нужно понять, где есть совпадение, а не читать сами строки, используйте `-l`.

```bash
rg -l 'rate_limited' logs
```

Должны появиться:

```text
logs/2026-02-20.log
logs/2026-02-21.log
logs/2026-02-22.log
```

## Подсчёт совпадений: `-c`

Если нужен не список строк, а количество совпадений по файлам, используйте `-c`.

```bash
rg -c 'PlayerConnected' logs
```

Должны появиться:

```text
logs/2026-02-20.log:5
logs/2026-02-21.log:4
logs/2026-02-22.log:6
```

## Фильтр по имени файла: `-g`

`rg` умеет искать не во всём подряд, а только в файлах по шаблону.

```bash
rg -n -g '*.log' 'Skin_Dragon'
```

Должны быть видны совпадения только в файлах `logs/*.log`.

Здесь `-g '*.log'` оставляет только `.log` файлы.

Наберите:

```bash
rg -l -g '*.txt' 'region=europe'
```

Должны появиться:

```text
players/europe/anna.txt
players/europe/ivan.txt
players/europe/max.txt
```

## Буквальный поиск: `-F`

По умолчанию `rg` понимает регулярные выражения. Если нужен точный текст как есть, без специальных символов, добавьте `-F`.

```bash
rg -F 'max+alt@mail.ru' logs
```

Должна найтись одна строка из `logs/2026-02-22.log`.

Здесь `+` — специальный символ для регулярных выражений, поэтому буквальный режим удобнее и надёжнее.

Наберите:

```bash
rg -F 'clan=dragons' players
```

Должны быть видны строки из профилей `anna` и `ivan`.

## Скрытые файлы: `--hidden`

По умолчанию `rg` пропускает скрытые файлы и скрытые папки. Если они нужны, добавьте `--hidden`.

```bash
rg 'draft-profile' .
```

Вывода быть не должно.

Теперь та же команда, но уже со скрытыми файлами:

```bash
rg --hidden 'draft-profile' .
```

Должна найтись строка из `players/.drafts/anna.tmp`.

## Шпаргалка

- `rg 'text' path` — искать по папке рекурсивно.
- `rg -n 'text' path` — показать номера строк.
- `rg -l 'text' path` — показать только имена файлов.
- `rg -c 'text' path` — посчитать совпадения по файлам.
- `rg -g '*.log' 'text'` — искать только в нужных файлах.
- `rg -F 'text' path` — искать буквальный текст.
- `rg --hidden 'text' path` — включить скрытые файлы.

## Напоследок

- `rg` называют `ripgrep`. Его написал Andrew Gallant, а сама утилита стала популярной потому, что сочетает скорость, рекурсивный поиск и удобные настройки по умолчанию.
- В отличие от простого `grep`, `rg` сразу уважает скрытые файлы и правила игнора. Поэтому для больших рабочих папок он обычно удобнее как основной инструмент поиска.

## Практика

13.1. Во всех дневных логах найдите буквальный текст `max+alt@mail.ru` и покажите номер строки.

   Должна найтись одна строка:

   - `logs/2026-02-22.log:28:...`

13.2. По всей папке найдите `draft-profile`, включая скрытые файлы, и покажите только имя файла.

   Должен появиться один файл:

   - `players/.drafts/anna.tmp`

13.3. Найдите `Skin_Dragon` только в `.log` файлах и посчитайте, сколько совпадений в каждом дневном логе.

   Должны появиться:

   - `logs/2026-02-20.log:2`
   - `logs/2026-02-21.log:1`
   - `logs/2026-02-22.log:3`
