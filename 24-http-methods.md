# 24. GET и POST: методы HTTP-запросов

В прошлом уроке вы видели статус-коды и заголовки. Теперь разберём методы — это то, что стоит в начале каждого запроса и говорит серверу, что именно вы хотите сделать.

## GET: получить данные

GET — самый распространённый метод. Браузер использует его каждый раз, когда вы открываете ссылку. Данные передаются прямо в URL через параметры.

Откройте DevTools (F12 → Network) и перейдите на:

```
https://httpbin.org/get?city=Caerleon&item=T6_BAG
```

В ответе найдите раздел `"args"` — там будут параметры, которые сервер получил:

```json
"args": {
  "city": "Caerleon",
  "item": "T6_BAG"
}
```

Параметры из URL видны прямо в адресной строке и сохраняются в истории браузера. Поэтому GET не используют для отправки паролей или чувствительных данных.

## POST: отправить данные

POST отправляет данные в теле запроса — они не видны в URL. Используется для создания данных, логина, загрузки файлов.

Чтобы увидеть POST в DevTools, нужно его спровоцировать. Откройте консоль браузера (та же F12, вкладка **Console**) и выполните:

```javascript
fetch('https://httpbin.org/post', {
  method: 'POST',
  headers: {'Content-Type': 'application/json'},
  body: JSON.stringify({city: 'Caerleon', price: 46991})
})
```

Переключитесь на вкладку **Network** — появится запрос `post`. Кликните на него и посмотрите:

- **Headers → Request Headers** — там будет `Content-Type: application/json`
- **Response** — в поле `"json"` сервер вернёт то, что вы отправили:

```json
"json": {
  "city": "Caerleon",
  "price": 46991
}
```

## Разница между GET и POST

| | GET | POST |
|--|-----|------|
| Данные | в URL | в теле запроса |
| Видимость | в адресной строке | скрыты |
| Кэшируется | да | нет |
| Использование | получить данные | отправить, создать |

## Смотрим заголовки запроса

Перейдите на:

```
https://httpbin.org/headers
```

Сервер вернёт только заголовки вашего запроса — те, что браузер отправил автоматически:

```json
"headers": {
  "Accept": "text/html,application/xhtml+xml...",
  "Accept-Language": "ru-RU,ru;q=0.9",
  "Host": "httpbin.org",
  "User-Agent": "Mozilla/5.0 (Windows NT 10.0...)"
}
```

Обратите внимание на `User-Agent` — по нему сервер понимает, что запрос пришёл из браузера Chrome на Windows. Когда запрос придёт из `curl`, там будет другое значение.

## Фильтр Fetch/XHR в DevTools

На обычном сайте в Network появляются десятки запросов — картинки, стили, скрипты. Чтобы видеть только запросы к API, нажмите фильтр **Fetch/XHR** над списком запросов.

Попробуйте: откройте любой сайт с авторизацией, войдите в аккаунт и посмотрите в Network с фильтром Fetch/XHR — там будет виден POST-запрос с вашими данными (но не паролем — он передаётся по HTTPS зашифрованным).

## Шпаргалка

- `GET` — получить данные, параметры в URL.
- `POST` — отправить данные, тело запроса скрыто.
- `Content-Type: application/json` — тип тела запроса.
- Fetch/XHR — фильтр в DevTools для API-запросов.
- `httpbin.org/get` — смотреть параметры GET.
- `httpbin.org/post` — смотреть тело POST.
- `httpbin.org/headers` — смотреть заголовки запроса.

## Напоследок

- Кроме GET и POST есть другие методы: PUT и PATCH — изменить данные, DELETE — удалить. В REST API они используются по смыслу: GET для чтения, POST для создания, PUT/PATCH для изменения, DELETE для удаления.
- Albion Online Data API использует только GET — данные только читаются, ничего не создаётся. Большинство публичных API только для чтения устроены так же.

## Практика

24.1. Откройте `https://httpbin.org/get?item=T6_BAG&city=Bridgewatch` и найдите в ответе раздел `"args"`.

   Должны появиться оба параметра: `"item": "T6_BAG"` и `"city": "Bridgewatch"`.

24.2. Выполните POST-запрос через консоль браузера на `https://httpbin.org/post` с любыми данными. Найдите их в ответе в поле `"json"`.

   Данные, которые вы отправили, должны появиться в теле ответа в разделе `"json"`.

24.3. Перейдите на `https://httpbin.org/headers` и найдите значение `User-Agent`.

   Должна появиться строка с названием вашего браузера и операционной системы.
