# 14. `sed`: быстро переписываем строки по шаблону

`grep` и `rg` помогают найти нужные строки. Следующая частая задача — быстро переписать найденный текст по шаблону, не открывая редактор. Для этого и нужен `sed`.

`sed` — потоковый редактор. Обычно он читает входной текст, применяет правило и печатает результат в терминал, не меняя исходный файл.

Перейдите в папку:

```bash
cd game_server_intro
```

Дальше все команды запускайте внутри неё.

## Базовая замена

Самая частая форма у `sed` такая:

```bash
sed 's/OLD/NEW/' FILE
```

- `s` — команда замены;
- `OLD` — что искать;
- `NEW` — на что заменить.

Например:

```bash
sed 's/region=/region: /' players/europe/anna.txt
```

Должна измениться только строка `region=europe`: вместо неё должна стать видна строка `region: europe`.

По умолчанию `sed` заменяет только первое совпадение в каждой строке.

Наберите:

```bash
sed 's/,/ | /' items/anna_inventory.csv
```

В каждой строке должна замениться только первая запятая.

## Все вхождения в строке: `g`

Если замена должна пройти не один раз, а по всей строке, добавьте флаг `g`.

```bash
sed 's/,/ | /g' items/anna_inventory.csv
```

Должны быть видны строки такого вида:

- `anna | Staff of Light | rare | 1`
- `anna | Health Potion | common | 8`
- `anna | Revive Scroll | epic | 1`

Наберите:

```bash
sed 's/ /_/g' players/europe/anna.txt
```

В строках, где есть пробелы, они должны замениться на `_`.

## Печатать только выбранные строки: `-n` и `p`

По умолчанию `sed` печатает весь поток. Если нужны только некоторые строки, включите `-n` и явно укажите, что печатать, через `p`.

```bash
sed -n '2,4p' players/europe/anna.txt
```

Должны остаться только строки:

- `region=europe`
- `level=38`
- `class=healer`

Наберите:

```bash
sed -n '/Skin_Dragon/p' logs/2026-02-20.log
```

Должны быть видны только строки с `Skin_Dragon`.

## Удалить строки из потока: `d`

`sed` умеет не только переписывать строки, но и выкидывать ненужные.

```bash
sed '/INFO/d' logs/2026-02-20.log
```

Должны остаться только строки с `ERROR` и `WARNING`.

Здесь команда `d` удаляет из потока все строки, где встретился шаблон `INFO`.

Наберите:

```bash
sed '1,2d' players/europe/anna.txt
```

Должны исчезнуть первые две строки профиля.

## Замена только в выбранных строках

У `sed` можно сначала выбрать строки, а потом применить к ним замену.

```bash
sed '/AuthFailed/s/ERROR/LOGIN_ERROR/' logs/2026-02-20.log
```

Должны измениться только строки с `AuthFailed`: в них `ERROR` должен стать `LOGIN_ERROR`.

Строки с `Disconnect` должны остаться с обычным `ERROR`.

Наберите:

```bash
sed '1,6s/=/: /' matches/match_1001.txt
```

В первых шести строках `=` должен замениться на `: `. Остальные строки должны остаться без изменений.

## Шпаргалка

- `sed 's/old/new/' file` — заменить первое совпадение в строке.
- `sed 's/old/new/g' file` — заменить все совпадения в строке.
- `sed -n '2,4p' file` — вывести только нужные строки.
- `sed '/text/d' file` — удалить строки с шаблоном.
- `sed '/text/s/old/new/' file` — заменить текст только в выбранных строках.

## Напоследок

- Название `sed` — сокращение от `stream editor`. Это одна из старых Unix-утилит для поточного редактирования текста.
- `sed -i` умеет менять файл на месте, но синтаксис у GNU `sed` и BSD/macOS `sed` отличается. Для начала безопаснее смотреть результат в терминале и только потом решать, нужно ли переписывать файл.
