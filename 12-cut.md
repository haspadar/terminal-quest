# 12. `cut`: вырезаем нужные поля

После `grep` и `rg` часто получается слишком много лишнего. Строку ты уже нашёл, но теперь хочется вытащить из неё только нужную часть: значение после `=`, победителя матча, первые столбцы CSV. Для этого и нужен `cut`.

Перейди в папку:

```bash
cd game_server_intro
```

## Разделитель и номер поля

Если строки устроены как `key=value`, удобнее всего брать разделитель `=` и второе поле.

```bash
grep '^winner=' matches/* | cut -d= -f2
```

Должно быть:
`john`
`team`
`ivan`
`kenji`

Здесь `-d=` задаёт разделитель, а `-f2` говорит “возьми второе поле”.

## Вырезать числа из `key=value`

```bash
grep '^level=' players/*/*.txt | cut -d= -f2
```

Должно быть: в выводе остаются только числа уровней без `level=`.

## Работать с CSV

В инвентарях разделитель уже другой, не `=`, а запятая.

```bash
cut -d, -f1,2 items/anna_inventory.csv
```

Должно быть:
`nickname,item`
`anna,Staff of Light`
`anna,Health Potion`
`anna,Revive Scroll`

## Использовать после поиска

`cut` особенно полезен в пайпе, когда нужные строки уже нашёл `grep`.

```bash
grep '^gold=' players/*/*.txt | cut -d= -f2
```

Должно быть: в выводе остаются только числовые значения золота.

## Шпаргалка

- `cut -d= -f2` взять значение после `=`.
- `cut -d, -f1,2` взять первые два столбца CSV.
- `grep 'x' ... | cut ...` сначала найти нужные строки, потом вырезать поле.
