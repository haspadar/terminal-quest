# 10. `xargs`: передаём список дальше

До этого момента команды либо находили список, либо выводили строки, либо вырезали из них нужные части. Но часто задача звучит так: список уже есть, теперь надо передать его другой команде как аргументы.

Для этого и нужен `xargs`.

Перейдите в папку:

```bash
cd game_server_intro
```

Дальше все команды запускайте внутри неё.

## Что делает `xargs`

`xargs` читает данные из `stdin`, собирает из них аргументы и запускает другую команду.

Именно для этого он и появился: одной команде удобно отдать поток строк, а другой команде нужен список аргументов. Также `xargs` защищает от слишком длинной командной строки, если элементов очень много.

Базовая форма:

```bash
... | xargs COMMAND
```

Например:

```bash
printf 'anna\nivan\nmax\n' | xargs
```

Должна появиться строка `anna ivan max`.

Если команду не указать, `xargs` по умолчанию передаст значения в `echo`.

Наберите:

```bash
printf 'anna\nivan\nmax\n' | xargs printf '%s\n'
```

Должны быть видны:

- `anna`
- `ivan`
- `max`

## Зачем он нужен после `find`

`find` умеет находить файлы, но следующая команда часто ждёт не поток строк, а список аргументов.

```bash
find players -name '*.txt' | xargs wc -l
```

Должны быть видны:

- `8 players/america/carlos.txt`
- `8 players/america/john.txt`
- `8 players/asia/kenji.txt`
- `8 players/asia/li.txt`
- `8 players/europe/anna.txt`
- `8 players/europe/ivan.txt`
- `8 players/europe/max.txt`
- `56 total`

Здесь `find` отдаёт пути, а `xargs` подставляет их в `wc -l` как обычные аргументы командной строки.

Наберите:

```bash
find players/europe -name '*.txt' | sort | xargs wc -c
```

Должны быть видны:

- `115 players/europe/anna.txt`
- `125 players/europe/ivan.txt`
- `113 players/europe/max.txt`
- `353 total`

## Передать список в другую фильтрацию

`xargs` хорошо сочетается не только с `wc`, но и с теми командами, которые вы уже знаете.

```bash
find players -name '*.txt' | sort | xargs cut -d= -f1 | sort | uniq
```

Должны быть видны:

- `clan`
- `class`
- `created_at`
- `email`
- `gold`
- `level`
- `nickname`
- `region`

Здесь `xargs` передаёт найденные файлы в `cut`, а дальше уже работает обычный пайп.

Наберите:

```bash
find players/europe -name '*.txt' | sort | xargs cat
```

Должны быть видны подряд все строки из `anna.txt`, `ivan.txt`, `max.txt`.

## По одному элементу: `-n 1`

Иногда полезно запускать команду не на всём списке сразу, а по одному элементу.

```bash
printf 'anna\nivan\nmax\n' | xargs -n 1 printf 'player: %s\n'
```

Должны быть видны:

- `player: anna`
- `player: ivan`
- `player: max`

Флаг `-n 1` говорит: передавать команде по одному аргументу за раз.

Наберите:

```bash
find logs -name '*.log' | sort | xargs -n 1 wc -l
```

Должны быть видны:

- `28 logs/2026-02-20.log`
- `28 logs/2026-02-21.log`
- `28 logs/2026-02-22.log`

## Когда `xargs` особенно полезен

`xargs` нужен там, где одна команда уже собрала список, а следующая команда хочет получить этот список как аргументы.

Часто используется в связках:

- `find ... | xargs wc ...`
- `find ... | xargs cut ...`
- `printf ... | xargs ...`

## Где с ним легко ошибиться

Обычный `xargs` разбивает ввод по пробелам. Поэтому с именами файлов, где есть пробелы, нужно быть осторожным.

Именно поэтому часто используют более безопасную связку:

```bash
find . -name '*.log' -print0 | xargs -0 command
```

В этом уроке таких файлов нет, примеры работают без доп. флагов.

## Шпаргалка

- `... | xargs command` передать список в команду как аргументы.
- `find ... | xargs wc -l` передать найденные файлы в `wc`.
- `find ... | xargs cut ...` передать найденные файлы в `cut`.
- `... | xargs -n 1 command` запускать команду по одному аргументу.

## Напоследок

1. Одна из причин появления `xargs` — ограничение на длину командной строки. Если список аргументов становится слишком длинным, `xargs` может разбить его на части и запускать команду несколько раз.

2. Исторически связка `find ... | xargs ...` была очень популярной. Позже у `find` появился вариант `-exec ... {} +`, который решает похожую задачу без отдельного `xargs`.
