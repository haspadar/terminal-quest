# 16. `eza`: современная замена `ls`

`ls` показывает файлы, но без цветов по типу, без git-статуса и без удобного дерева. `eza` делает то же самое, но с подсветкой и деревом в одной команде.

> `exa` — оригинальный проект, разработка которого остановилась в 2023 году. `eza` — его активно поддерживаемый форк. Команды у них идентичны.

Перейдите в папку:

```bash
cd game_server_intro
```

## Базовый вывод

```bash
eza
```

Должны появиться папки и файлы с подсветкой по типу: папки — одним цветом, файлы — другим.

## Подробный список

```bash
eza -l
```

Должны появиться права, размер, дата и имя. В отличие от `ls -l`, размер показывается в читаемом виде по умолчанию (`1k`, `416B`).

## Скрытые файлы

```bash
eza -la
```

Должны появиться `.cache`, `.env.example`, `.notes` вместе с остальными файлами.

## Дерево

```bash
eza --tree --level=2
```

Должна появиться структура папок до второго уровня — как `tree`, но с подсветкой.

Попробуйте с уровнем 3:

```bash
eza --tree --level=3
```

Должны появиться файлы внутри `logs/`, `chat/`, `players/` и других подпапок.

## Git-статус

Внутри git-репозитория `eza` умеет показывать статус файлов:

```bash
cd ..
eza -l --git
```

Должна появиться колонка с git-статусом рядом с именем файла. Незакоммиченные изменения обозначаются буквой `M`, новые файлы — `N`.

## Шпаргалка

- `eza` — базовый список с подсветкой.
- `eza -l` — подробный список.
- `eza -la` — подробный список со скрытыми файлами.
- `eza --tree --level=N` — дерево на N уровней.
- `eza -l --git` — список с git-статусом.

## Напоследок

- `eza` написана на Rust — скорость на больших папках заметно выше `ls`.
- Если в терминале установлен Nerd Font, добавьте флаг `--icons` — перед каждым файлом появится иконка по типу.

## Практика

16.1. Выведите подробный список файлов в `game_server_intro`, включая скрытые.

   Должны появиться `.cache`, `.env.example` и `.notes` вместе с остальными.

16.2. Покажите дерево `game_server_intro` на два уровня.

   Должна быть видна структура подпапок, но не содержимое файлов.

16.3. Перейдите в корень репозитория и выведите список файлов с git-статусом. Создайте любой новый файл и снова выведите список.

   В колонке git-статуса должен появиться маркер `N` рядом с новым файлом.
