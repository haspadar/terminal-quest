# 12. `grep`: ищем строки внутри файлов

`find` отвечает на вопрос: в каком файле искать. Следующий вопрос обычно такой: где внутри файла есть нужная строка. Для этого и нужен `grep`.

Перейдите в папку:

```bash
cd game_server_intro
```

Дальше все команды запускайте внутри неё.

## Базовая форма

У `grep` две главные части:

```bash
grep 'PATTERN' FILES
```

- `PATTERN` — что искать;
- `FILES` — в каких файлах искать.

Например:

```bash
grep 'ERROR' logs/2026-02-20.log
```

Должны найтись строки с ошибками из `logs/2026-02-20.log`.

Это самый простой режим: `grep` ищет точный текст и выводит подходящие строки целиком.

Наберите:

```bash
grep 'PlayerConnected' logs/2026-02-20.log
```

Должны быть видны только строки с подключением игроков.

## Поиск без учёта регистра: `-i`

Если в шаблоне важны буквы, но не важен регистр, добавьте `-i`.

```bash
grep -i 'authfailed' logs/2026-02-20.log
```

Должны найтись строки с `AuthFailed`, хотя в шаблоне всё написано маленькими буквами.

Наберите:

```bash
grep -i 'warning' logs/2026-02-20.log
```

Должны быть видны строки с `WARNING`.

## Убрать строки по шаблону: `-v`

Иногда нужно не оставить совпадения, а наоборот убрать их. Для этого есть `-v`.

```bash
grep -v 'INFO' logs/2026-02-20.log
```

Должны остаться только строки с `ERROR` и `WARNING`.

`grep -v` полезен как быстрый фильтр: сначала убрали лишнее, потом уже смотрите на остаток.

Наберите:

```bash
grep -v 'region=' players/europe/anna.txt
```

Должны остаться все строки профиля, кроме `region=europe`.

## Начало и конец строки: `^` и `$`

`grep` понимает регулярные выражения. Самые полезные якоря в начале:

- `^` — начало строки;
- `$` — конец строки.

Например:

```bash
grep '^clan=' players/europe/anna.txt
```

Должна остаться строка `clan=dragons`.

Здесь `grep` ищет не просто `clan`, а именно строки, которые начинаются с `clan=`.

Наберите:

```bash
grep 'europe$' players/*/*.txt
```

Должны быть видны только строки `region=europe` из трёх европейских профилей.

## Номера строк: `-n`

Когда совпадений много, полезно сразу видеть номера строк.

```bash
grep -n 'AuthFailed' logs/*.log
```

Должны быть видны имя файла, номер строки и сама строка.

Например:

```text
logs/2026-02-20.log:2:...
```

## Только имена файлов: `-l`

Иногда сама строка не нужна. Нужно только понять, в каких файлах есть совпадение.

```bash
grep -l '^clan=dragons' players/*/*.txt
```

Должны появиться:

```text
players/europe/anna.txt
players/europe/ivan.txt
```

## Подсчёт совпадений: `-c`

Если нужен не список строк, а число, поможет `-c`.

```bash
grep -c 'ERROR' logs/*.log
```

Должны появиться:

```text
logs/2026-02-20.log:6
logs/2026-02-21.log:10
logs/2026-02-22.log:6
```

## Один из нескольких шаблонов: `-E`

Когда нужен не один текст, а несколько вариантов, удобно включить расширенные регулярные выражения через `-E`.

```bash
grep -E 'ERROR|WARNING' logs/2026-02-20.log
```

Должны остаться строки, где есть `ERROR` или `WARNING`.

Символ `|` здесь значит: подойдёт левый шаблон или правый.

Наберите:

```bash
grep -E 'wrong_password|rate_limited' logs/2026-02-20.log
```

Должны быть видны строки, где причина ошибки — `wrong_password` или `rate_limited`.

## Шпаргалка

- `grep 'text' file` — искать текст.
- `grep -i 'text' file` — не учитывать регистр.
- `grep -v 'text' file` — убрать строки с совпадением.
- `grep -n 'text' files` — показать номера строк.
- `grep -l 'text' files` — показать только имена файлов.
- `grep -c 'text' files` — посчитать совпадения по файлам.
- `grep '^text' file` — искать в начале строки.
- `grep 'text$' file` — искать в конце строки.
- `grep -E 'a|b' file` — искать один из нескольких шаблонов.

## Напоследок

- Название `grep` пришло из команды `g/re/p` в редакторе `ed`: global, regular expression, print. Отдельная утилита появилась в Unix Version 4 в 1973 году; её написал Ken Thompson.
- Для простого текстового поиска `grep` обычно очень быстрый: он читает текст потоком и хорошо оптимизирован под регулярные выражения. Но искать по всей папке удобнее следующим инструментом — `rg`.

12.1. Найдите во всех дневных логах строки, где есть `ERROR`, но нет `AuthFailed`.

   Должны остаться только события `Disconnect` из трёх файлов `logs/*.log`.

   - `players/europe/ivan.txt`
   - `players/europe/max.txt`

   Должно появиться число `24`.
