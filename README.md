# Terminal Quest

Курс по командной строке.

Научимся анализировать данные на практике:
- понимать структуру каталогов, 
- находить файлы, 
- искать по содержимому, 
- считать найденное, 
- сортировать,
- собирать пайплайны

Предполагается, что ты уже знаком с основами по командной строке:
- что такое shell и зачем он нужен (запуск команд, аргументы, флаги)
- как перемещаться по файловой системе (`pwd`, `cd`, `ls`)
- как читать и редактировать файлы (`cat`, `less`, `nano` или `vim`)
- что такое потоки и пайпы (`stdin`, `stdout`, `stderr`, оператор `|`)
- как работают базовые команды поиска по тексту (`grep`, простые шаблоны).
- что такое регулярные выражения и как применять их на практике.

Примеры в курсе рассчитаны на Linux и `fish`.
Лучше всего проходить курс в Ubuntu.
На macOS тоже можно, но часть утилит может отличаться.

Для первых уроков используется архив `game_server_intro.zip`.

## Как скачать

Сначала установи нужные инструменты:

```bash
sudo apt update
sudo apt install fish git nodejs npm unzip
```

Потом скачай репозиторий:

```bash
git clone <URL_РЕПОЗИТОРИЯ>
cd terminal-quest
```

## Codex

Если хочешь использовать агента как ассистента по задачам, запускай `codex` в отдельной вкладке терминала.
В одной вкладке идёт урок, в другой работает Codex.

1. Установить Codex CLI:

```bash
npm i -g @openai/codex
```

2. Перейти в каталог репозитория:

```bash
cd terminal-quest
```

3. Запустить Codex:

```bash
codex
```

При первом запуске нужно будет войти в аккаунт.

Если OpenAI недоступен напрямую, заранее включи VPN.

Внутри этого репозитория для помощи с уроками есть локальный skill `mentor`.
Отдельно запускать его не нужно. Просто обратись к нему в начале сообщения.

Mentor подсказывает постепенно и не выдаёт полное решение сразу.

Формат задач такой: `урок.номер задачи`.
Например, `1.4` означает: урок 1, задача 4.

Пример:

```text
mentor, 1.4
моя команда: tree logs -L 2
что не так?
```

Ответ:

```text
Ты смотришь только внутрь `logs`.
Здесь задача про весь набор данных.
Проверь, нужен ли тебе тут аргумент `logs`.
```
