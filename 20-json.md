# 20. JSON: как программы хранят и передают данные

Программы постоянно обмениваются данными — браузер получает список товаров с сервера, игра сохраняет настройки, API возвращает результат запроса. Для этого нужен единый формат, который понимает любой язык программирования. Таким форматом стал JSON.

## Что такое JSON

JSON — это текстовый формат для хранения структурированных данных. Расшифровывается как JavaScript Object Notation, но сегодня его используют везде: Python, Go, Java, PHP — любой язык умеет читать и писать JSON.

Посмотрите на пример:

```json
{
  "name": "GameHub-01",
  "region": "eu-west",
  "max_players": 250,
  "allow_guest": false
}
```

Это те же данные, что и в `server.ini`, но в JSON-формате. Легко читается и человеком, и программой.

## Объект: фигурные скобки

Основная единица JSON — **объект**. Это набор пар «ключ: значение» в фигурных скобках:

```json
{
  "id": "desert",
  "mode": "pvp",
  "size": "large"
}
```

Правила:
- Ключ — всегда строка в **двойных** кавычках.
- После ключа — двоеточие.
- Между парами — запятая, но **не после последней**.
- Одинарные кавычки и комментарии не допускаются.

## Типы значений

Значения бывают разных типов:

```json
{
  "name": "anna",
  "level": 42,
  "price": 19.99,
  "active": true,
  "clan": null
}
```

- `"anna"` — строка, всегда в двойных кавычках.
- `42`, `19.99` — числа, без кавычек.
- `true`, `false` — булевы значения, без кавычек.
- `null` — отсутствие значения, без кавычек.

## Массив: квадратные скобки

Массив — это упорядоченный список значений в квадратных скобках:

```json
["pvp", "pve", "ranked"]
```

Массив может содержать объекты — это самый частый случай на практике:

```json
[
  {"id": "desert", "mode": "pvp"},
  {"id": "arena",  "mode": "ranked"}
]
```

## Вложенность

Объекты и массивы вкладываются друг в друга на любую глубину. Посмотрите на файл `configs/maps.json`:

```fish
cat configs/maps.json
```

Вы увидите объект с ключом `"maps"`, значение которого — массив объектов:

```json
{
  "maps": [
    {"id": "desert",    "mode": "pvp",    "size": "large"},
    {"id": "forest",    "mode": "pvp",    "size": "medium"},
    {"id": "arena",     "mode": "ranked", "size": "small"},
    {"id": "ice_caves", "mode": "pve",    "size": "medium"}
  ]
}
```

## Частые ошибки

Если JSON содержит ошибку, программа не сможет его прочитать. Самые распространённые:

| Ошибка | Правильно |
|--------|-----------|
| `{'id': 'desert'}` | `{"id": "desert"}` — только двойные кавычки |
| `{"id": desert}` | `{"id": "desert"}` — строки в кавычках |
| `{"size": "large",}` | `{"size": "large"}` — запятая перед `}` недопустима |
| `// комментарий` | убрать — в JSON нет комментариев |

## Поиск внутри JSON через rg

JSON — это обычный текст, поэтому `rg` и `grep` работают с ним напрямую.

Найдите все карты с режимом `pvp`:

```fish
rg 'pvp' configs/maps.json
```

Должны появиться две строки — `desert` и `forest`.

Найдите все значения `medium` по всей папке конфигов:

```fish
rg 'medium' configs/
```

Должны найтись две строки из `maps.json`.

Это работает, пока нужно найти конкретную строку. Но что если нужно достать только значение одного поля из каждого объекта, или отфильтровать карты по режиму? `rg` для этого не предназначен. В следующем уроке будет `jq` — инструмент, который понимает структуру JSON.

## Шпаргалка

- `{...}` — объект: набор пар «ключ: значение».
- `[...]` — массив: упорядоченный список.
- Ключи и строки — только в двойных кавычках.
- `true`, `false`, `null`, числа — без кавычек.
- Запятая между элементами, но не после последнего.
- `cat file.json` — просмотреть содержимое.
- `rg 'pattern' file.json` — поиск по содержимому.

## Напоследок

- JSON появился в начале 2000-х как упрощённая альтернатива XML. Синтаксис взят из JavaScript, но сегодня JSON используется во всех языках и стал фактическим стандартом для API и конфигов.
- JSON не поддерживает комментарии намеренно — это упрощает парсинг. Для конфигов с комментариями используют TOML или YAML (например, `starship.toml` из урока 18).

## Практика

20.1. Откройте `configs/maps.json` с помощью `cat` и найдите, сколько карт в файле.

   Должны быть видны четыре объекта: `desert`, `forest`, `arena`, `ice_caves`.

20.2. Найдите в `configs/maps.json` все карты размера `medium`.

   Должны найтись две строки: `forest` и `ice_caves`.

20.3. Попробуйте с помощью `rg` найти только названия карт — значения ключа `"id"`.

   Выполните `rg '"id"' configs/maps.json`. Должны появиться четыре строки — по одной на каждую карту. Обратите внимание: `rg` возвращает строку целиком, а не только значение. Вытащить только `"desert"` из `"id": "desert"` через `rg` уже неудобно — для этого нужен `jq`.
