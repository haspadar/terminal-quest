# 06. Переменные: сохраняем и подставляем

После `find`, `wc`, `sort` и `uniq` появляется задача: длинный путь или список файлов приходится набирать снова и снова. Хочется сохранить значение и подставлять в команды.

Для этого в shell есть переменные. Всё крутится вокруг текста — переменная хранит строку или список строк, которые подставляются в команду.

## Запись значения: `set`

В `fish` переменная задаётся через `set` (не `name="Alex"`, как в `bash`):

```bash
set name Alex
```

Перейдите в папку и сохраните путь в переменную:

```bash
cd game_server_intro
set folder players/europe
find $folder -name '*.txt'
```

Должны найтись:
- `players/europe/anna.txt`
- `players/europe/ivan.txt`
- `players/europe/max.txt`

Здесь `$folder` подставляет путь в команду.

Попробуйте сами:

```bash
set target chat/global_chat.log
wc -l $target
```

Должно появиться число `7`.

## Сохранить вывод команды

Переменная может хранить результат другой команды:

```bash
set logs (find logs -name '*.log' | sort)
wc -c $logs
```

Должны быть видны:
- `1952 logs/2026-02-20.log`
- `2066 logs/2026-02-21.log`
- `1975 logs/2026-02-22.log`
- `5993 total`

Скобки `(...)` значат: выполни команду, сохрани вывод.

```bash
set files (find players/europe items -type f | sort)
wc -l $files
```

Должны быть видны 11 файлов и `53 total`.

В переменной лежит список путей — `fish` подставляет его как несколько аргументов.

## Когда полезно

- путь или имя используется несколько раз;
- команда длинная, и её результат хочется сохранить;
- один и тот же список нужно передать в другую команду.

## Шпаргалка

- `set name value` — записать значение
- `$name` — подставить значение
- `set name (command)` — сохранить вывод команды
- в `fish` переменная хранит и одно значение, и список

## Напоследок

1. В ранних Unix shell не было переменных. Идея выросла из задачи: передавать параметр, не набирая каждый раз.
2. Shell появился раньше многих языков программирования — поэтому здесь нет типизации, всё крутится вокруг текста.

## Практика

   - Должны найтись: `players/europe/anna.txt`, `players/europe/ivan.txt`, `players/europe/max.txt`

6.2. Сохраните путь `chat/global_chat.log` в переменную и используйте в `wc -l`.
   - Должно появиться число `7`

6.3. Сохраните отсортированный список `.log` файлов из `logs` и используйте в `wc -c`.
   - Должны быть видны: `1952 logs/2026-02-20.log`, `2066 logs/2026-02-21.log`, `1975 logs/2026-02-22.log`, `5993 total`

6.4. Сохраните отсортированный список файлов из `players/europe` и `items`, передайте в `wc -l`.
   - Должны быть видны: `4 items/anna_inventory.csv`, `4 items/carlos_inventory.csv`, `5 items/ivan_inventory.csv`, `4 items/john_inventory.csv`, `4 items/kenji_inventory.csv`, `4 items/li_inventory.csv`, `4 items/max_inventory.csv`, `8 players/europe/anna.txt`, `8 players/europe/ivan.txt`, `8 players/europe/max.txt`, `53 total`
