# 06. переменные: сохраняем и подставляем

После `find`, `wc`, `sort` и `uniq` появляется задача: длинный путь или список файлов приходится набирать снова и снова. Хочется сохранить значение и подставлять в команды.

Для этого в shell есть переменные.

## Что хранит переменная

В shell всё крутится вокруг текста. Переменная хранит строку или список строк, которые можно подставить в команду.

Shell — не типизированный язык. Нет отдельных `int`, `string` и `bool`. Значения просто уходят в аргументы команд.

## `set`: записать значение

В `fish` переменная задаётся через `set`. Нет записи `name="Alex"`, как в `bash`.

Самая простая форма выглядит так:

```bash
set name Alex
```

Перейдите в папку:

```bash
cd game_server_intro
```

Сначала сохраним путь в переменную.

```bash
set folder players/europe
find $folder -name '*.txt'
```

Должны найтись:

- `players/europe/anna.txt`
- `players/europe/ivan.txt`
- `players/europe/max.txt`

Здесь переменная `folder` хранит путь, а `$folder` подставляет его в `find`.

Наберите:

```bash
set target chat/global_chat.log
wc -l $target
```

Должно появиться число `7` для `chat/global_chat.log`.

Та же идея работает с файлами и папками: сохранили значение — используйте как аргумент.

## Запись через `set`

В `fish` пробелы работают иначе. Команда `set` отвечает за создание переменной.

Поэтому правильно так:

```bash
set folder players/europe
```

А не так:

```bash
folder = players/europe
```

Вторая запись в `fish` не работает как присваивание.

## Сохранить вывод команды

Переменная может хранить не только готовый текст, но и результат другой команды.

```bash
set logs (find logs -name '*.log' | sort)
wc -c $logs
```

Должны быть видны:

- `1952 logs/2026-02-20.log`
- `2066 logs/2026-02-21.log`
- `1975 logs/2026-02-22.log`
- `5993 total`

Скобки `(...)` значат: выполни команду, сохрани вывод в переменную.

Наберите:

```bash
set files (find players/europe items -type f | sort)
wc -l $files
```

Должны быть видны:

- `4 items/anna_inventory.csv`
- `4 items/carlos_inventory.csv`
- `5 items/ivan_inventory.csv`
- `4 items/john_inventory.csv`
- `4 items/kenji_inventory.csv`
- `4 items/li_inventory.csv`
- `4 items/max_inventory.csv`
- `8 players/europe/anna.txt`
- `8 players/europe/ivan.txt`
- `8 players/europe/max.txt`
- `53 total`

В переменной лежит список путей — `fish` подставляет его в команду как несколько аргументов.

Переменную удобно подставлять в путь. Например, `~/projects/$project`.

## Когда переменная особенно полезна

Переменные полезны в типичных ситуациях:

- путь или имя приходится использовать несколько раз;
- команда получилась длинной, и её результат хочется сохранить;
- один и тот же список файлов потом нужно передать в другую команду.

Поэтому тема переменных идёт после `find`, `wc`, `sort` и `uniq`.

## Шпаргалка

- `set name value` записать значение в переменную.
- `$name` подставить значение переменной в команду.
- `set name (command)` сохранить вывод команды.
- в `fish` переменная может хранить и одно значение, и список значений.

## Напоследок

1. В ранних версиях Unix shell не имел переменных. Идея выросла из простой задачи: передавать параметр программам, не набирая его каждый раз.

2. Shell появился раньше многих привычных языков программирования — поэтому переменные здесь не типизированы, а всё крутится вокруг текста и аргументов команд.
