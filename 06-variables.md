# 06. переменные: сохраняем и подставляем

После `find`, `wc`, `sort` и `uniq` быстро появляется одна и та же боль: длинный путь, кусок команды или список файлов приходится набирать снова и снова. Хочется сохранить значение один раз и потом подставлять его в знакомые команды.

Для этого в shell есть переменные.

## Что хранит переменная

В shell почти всё крутится вокруг текста. Переменная хранит строку или список строк, а потом это значение можно подставить в команду.

Это не типизированный язык программирования. Здесь нет отдельных `int`, `string` и `bool`. Для shell это просто значения, которые потом уходят в аргументы команд.

## `set`: записать значение

В `fish` переменная задаётся через `set`. Это важный момент: здесь нет записи вроде `name="Alex"`, как в `bash`.

Самая простая форма выглядит так:

```bash
set name Alex
```

Перейди в папку:

```bash
cd game_server_intro
```

Сначала сохраним в переменную обычный путь.

```bash
set folder players/europe
find $folder -name '*.txt'
```

Должны найтись:

- `players/europe/anna.txt`
- `players/europe/ivan.txt`
- `players/europe/max.txt`

Здесь переменная `folder` хранит путь, а `$folder` подставляет его в `find`.

Набери:

```bash
set target chat/global_chat.log
wc -l $target
```

Должно появиться число `7` для `chat/global_chat.log`.

Та же идея работает и с файлами, и с папками: сохранил значение, потом использовал его как аргумент команды.

## Почему важна запись через `set`

В `fish` пробелы в присваивании работают не так, как в языках программирования. Здесь сама команда `set` отвечает за создание переменной.

Поэтому правильно так:

```bash
set folder players/europe
```

А не так:

```bash
folder = players/europe
```

Вторая запись для `fish` просто не имеет смысла как присваивание.

## Сохранить вывод команды

Переменная может хранить не только готовый текст, но и результат другой команды.

```bash
set logs (find logs -name '*.log' | sort)
wc -c $logs
```

Должны быть видны:

- `1952 logs/2026-02-20.log`
- `2066 logs/2026-02-21.log`
- `1975 logs/2026-02-22.log`
- `5993 total`

Скобки `(...)` здесь значат: сначала выполни команду, потом сохрани её вывод в переменную.

Набери:

```bash
set files (find players/europe items -type f | sort)
wc -l $files
```

Должны быть видны:

- `4 items/anna_inventory.csv`
- `4 items/carlos_inventory.csv`
- `5 items/ivan_inventory.csv`
- `4 items/john_inventory.csv`
- `4 items/kenji_inventory.csv`
- `4 items/li_inventory.csv`
- `4 items/max_inventory.csv`
- `8 players/europe/anna.txt`
- `8 players/europe/ivan.txt`
- `8 players/europe/max.txt`
- `53 total`

Здесь в переменной уже лежит не одно значение, а целый список путей. `fish` умеет подставлять такой список в команду как несколько аргументов сразу.

Переменную удобно подставлять и прямо в путь. Например, если в переменной лежит имя папки или проекта, из него можно собрать что-то вроде `~/projects/$project`.

## Когда переменная особенно полезна

Переменные хороши в трёх типичных ситуациях:

- путь или имя приходится использовать несколько раз;
- команда получилась длинной, и её результат хочется сохранить;
- один и тот же список файлов потом нужно передать в другую команду.

Именно поэтому тема переменных идёт сейчас, после `find`, `wc`, `sort` и `uniq`: уже есть что сохранять и куда подставлять.

## Шпаргалка

- `set name value` записать значение в переменную.
- `$name` подставить значение переменной в команду.
- `set name (command)` сохранить вывод команды.
- в `fish` переменная может хранить и одно значение, и список значений.

## Напоследок

1. В самых ранних версиях Unix shell вообще не имел полноценной системы переменных.
Идея переменных выросла из практической задачи: как передавать параметр программам и не печатать его каждый раз заново.

2. Shell появился раньше многих привычных языков программирования.
Поэтому переменные здесь не типизированы, а сама работа крутится вокруг текста и аргументов команд.
