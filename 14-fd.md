# 14. `fd`: быстрый поиск файлов

`find` работает, но синтаксис громоздкий: `find . -name "*.log" -type f`. `fd` решает ту же задачу короче и быстрее.

Перейдите в папку:

```bash
cd game_server_intro
```

## Базовый поиск

```bash
fd 'log'
```

Должны найтись все файлы и папки, в имени которых есть `log`: и папка `logs/`, и файлы внутри неё, и `*.log` из `chat/`.

`fd` ищет рекурсивно по умолчанию и не требует указывать `.` или `-name`.

## Поиск по расширению

```bash
fd -e log
```

Должны появиться:

```
chat/clan_dragons.log
chat/global_chat.log
logs/2026-02-20.log
logs/2026-02-21.log
logs/2026-02-22.log
```

## Только файлы или только папки

```bash
fd -t f 'log'
fd -t d 'log'
```

Первая команда — только файлы. Вторая — только папки: должна остаться одна `logs/`.

## Скрытые файлы

По умолчанию `fd` пропускает скрытые файлы. Флаг `-H` включает их:

```bash
fd -H '^\.'
```

Должны появиться:

```
.cache/
.env.example
.notes
players/.drafts/
```

## Шпаргалка

- `fd 'pattern'` — поиск по имени рекурсивно.
- `fd -e log` — только файлы с расширением `.log`.
- `fd -t f 'pattern'` — только файлы.
- `fd -t d 'pattern'` — только папки.
- `fd -H 'pattern'` — включить скрытые файлы.

## Напоследок

- `fd` написан на Rust и на больших деревьях работает заметно быстрее стандартного `find`.
- `fd` по умолчанию уважает `.gitignore` — не ищет в папках, которые исключены из репозитория.

## Практика

14.1. Найдите все `.log` файлы в `game_server_intro`.

   Должны появиться пять файлов: три из `logs/` и два из `chat/`.

14.2. Найдите скрытые файлы в `game_server_intro`.

   Должны появиться `.cache/`, `.env.example`, `.notes` и `players/.drafts/`.

14.3. Найдите только директории, имя которых содержит `logs`.

   Должна появиться одна папка: `logs/`.
