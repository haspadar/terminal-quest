# 22. `jq` и pipe: анализируем данные из нескольких файлов

В прошлом уроке `jq` работал с одним файлом. Но данные за три дня лежат в трёх папках, а предметов несколько. Здесь разберём, как объединять файлы через pipe и строить запросы, которые дают реальные ответы: где дешевле купить, где дороже продать, как менялась цена по дням.

Перейдите в папку:

```fish
cd albion/prices
```

## Объединить несколько файлов: cat + jq -s

`cat` выдаёт файлы один за другим. Если передать это в `jq`, он увидит несколько JSON-массивов подряд — и запутается. Флаг `-s` («slurp») решает проблему: он читает весь ввод целиком и оборачивает в один большой массив.

```fish
cat 2026-03-24/T6_BAG.json 2026-03-25/T6_BAG.json 2026-03-26/T6_BAG.json | jq -s 'add'
```

`add` склеивает массив массивов в один плоский массив. Должны появиться 21 объект — по 7 городов за каждый из трёх дней.

Через glob — короче:

```fish
cat 2026-03-2*/T6_BAG.json | jq -s 'add'
```

## Динамика по дням: как менялась цена в одном городе

Отфильтруйте по городу и отсортируйте по дате:

```fish
cat 2026-03-2*/T6_BAG.json | jq -s 'add | sort_by(.date) | .[] | select(.city == "Caerleon") | "\(.date)  \(.sell_price_min)"' -r
```

Должны появиться три строки:

```
2026-03-24  48200
2026-03-25  46991
2026-03-26  46991
```

`"\(.date)  \(.sell_price_min)"` — строка с подстановкой значений. Флаг `-r` убирает кавычки.

Попробуйте то же самое для Fort Sterling — там цена резко упала 26 марта:

```fish
cat 2026-03-2*/T6_BAG.json | jq -s 'add | sort_by(.date) | .[] | select(.city == "Fort Sterling") | "\(.date)  \(.sell_price_min)"' -r
```

Должно появиться:

```
2026-03-24  62900
2026-03-25  61200
2026-03-26  42173
```

## Группировка: group_by

`group_by(.field)` собирает элементы в группы по значению поля — получается массив массивов. Сгруппируйте по городу и посмотрите историю цен:

```fish
cat 2026-03-2*/T6_BAG.json | jq -s '
  add |
  group_by(.city) |
  .[] |
  {city: .[0].city, days: [.[] | {date, price: .sell_price_min}]}
'
```

Для каждого города появится объект с массивом `days`. Удобно видеть всю динамику сразу.

## Арбитраж: где купить дёшево и продать дорого

`min_by` и `max_by` находят объект с минимальным и максимальным значением поля. Объедините результаты через `+` и посчитайте прибыль:

```fish
cat 2026-03-26/T6_BAG.json | jq '
  (min_by(.sell_price_min) | {buy_city: .city, buy: .sell_price_min}) +
  (max_by(.sell_price_min) | {sell_city: .city, sell: .sell_price_min}) |
  . + {profit: (.sell - .buy)}
'
```

Должно появиться:

```json
{
  "buy_city": "Thetford",
  "buy": 42000,
  "sell_city": "Bridgewatch",
  "sell": 59956,
  "profit": 17956
}
```

Тот же запрос для другого предмета — просто меняем файл:

```fish
cat 2026-03-26/T6_METALBAR.json | jq '
  (min_by(.sell_price_min) | {buy_city: .city, buy: .sell_price_min}) +
  (max_by(.sell_price_min) | {sell_city: .city, sell: .sell_price_min}) |
  . + {profit: (.sell - .buy)}
'
```

## Сравнить несколько предметов сразу

Объедините все файлы за один день, сгруппируйте по предмету и найдите самый дешёвый город для каждого:

```fish
cat 2026-03-26/*.json | jq -s '
  add |
  group_by(.item_id) |
  .[] |
  {
    item: .[0].item_id,
    cheapest_city: (min_by(.sell_price_min) | .city),
    price: (min_by(.sell_price_min) | .sell_price_min)
  }
'
```

Должны появиться три строки — по одной на каждый предмет с самым дешёвым городом.

## Шпаргалка

- `cat files | jq -s 'add'` — объединить несколько JSON-массивов в один.
- `sort_by(.field)` — сортировать массив по полю.
- `group_by(.field)` — сгруппировать по значению поля.
- `select(.city == "X")` — оставить только нужные элементы.
- `min_by(.field)` / `max_by(.field)` — элемент с мин/макс значением.
- `. + {key: value}` — добавить поле к объекту.
- `"\(.field1)  \(.field2)"` — строка с подстановкой значений.
- `-r` — вывод без кавычек.

## Напоследок

- `jq` исполняет фильтры в потоке — он не грузит весь файл в память целиком. Поэтому `cat` + `jq -s` работает даже на больших файлах.
- Когда запрос становится длинным, его удобно вынести в файл: `jq -f query.jq data.json`. Так же, как скрипт на shell.
