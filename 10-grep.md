# 10. `grep`: ищем строки внутри файлов

`find` отвечает на вопрос “в каком файле искать”. Следующий вопрос обычно такой: “а где внутри файла нужная строка?” Для этого и нужен `grep`.

Перейдите в папку:

```bash
cd game_server_intro
```

## Базовый поиск

```bash
grep 'ERROR' logs/2026-02-20.log
```

Должно быть: находятся строки с ошибками из `logs/2026-02-20.log`.

Это самый простой режим: ищем точный текст в одном файле.

## Номера строк

Когда совпадений много, полезно сразу видеть номера строк.

```bash
grep -n 'AuthFailed' logs/*.log
```

Должно быть: в выводе есть имя файла, номер строки и сама строка.

Например:
`logs/2026-02-20.log:2:...`

## Только имена файлов

Иногда строка сама не нужна. Нужно только понять, в каких файлах есть совпадение.

```bash
grep -l 'clan=dragons' players/*/*.txt
```

Должно быть:
`players/america/john.txt`
`players/europe/anna.txt`
`players/europe/ivan.txt`

## Подсчёт совпадений

Если нужен не список строк, а число, поможет `-c`.

```bash
grep -c 'ERROR' logs/*.log
```

Должно быть:
`logs/2026-02-20.log:6`
`logs/2026-02-21.log:10`
`logs/2026-02-22.log:6`

## Шпаргалка

- `grep 'text' file` искать текст.
- `grep -n 'text' files` показать номера строк.
- `grep -l 'text' files` показать только имена файлов.
- `grep -c 'text' files` посчитать совпадения по файлам.

## Практика

1. Найдите все строки с `ERROR` во всех файлах из `logs`.

Должно быть: совпадения находятся во всех трёх дневных логах.

2. Найдите строки с `AuthFailed` и покажите номера строк.

Должно быть: в выводе есть строки вроде `logs/2026-02-20.log:2:...`.

3. Покажите только имена файлов, где встречается `clan=dragons`.

Должно быть:
`players/america/john.txt`
`players/europe/anna.txt`
`players/europe/ivan.txt`

4. Посчитайте, сколько раз `ERROR` встречается в каждом дневном логе.

Должно быть:
`logs/2026-02-20.log:6`
`logs/2026-02-21.log:10`
`logs/2026-02-22.log:6`
