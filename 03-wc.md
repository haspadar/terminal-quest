# 03. `wc`: считаем строки, слова и байты

После `find` быстро хочется не только найти что-то, но и оценить масштаб. Сколько строк в логах? Сколько сообщений в чатах? Сколько весят файлы? Сколько файлов вообще нашлось?

Для этого и нужен `wc`.

Перейди в папку:

```bash
cd game_server_intro
```

## Название — не про туалет

`wc` расшифровывается как `word count`, то есть “подсчёт слов”. Но на практике команду используют не только для слов: чаще всего для строк, байтов и быстрых подсчётов через pipe.

## Синтаксис

Самый простой вид такой:

```bash
wc файл
```

Например:

```bash
wc logs/2026-02-20.log
```

Вывод:

```text
28 183 1952 logs/2026-02-20.log
```

Без флагов `wc` показывает сразу несколько чисел. Обычно это:
- строки;
- слова;
- байты.

Если нужен только один конкретный подсчёт, лучше сразу взять флаг.

## Считать строки: `-l`

Чаще всего `wc` используют именно для строк.

```bash
wc -l logs/*.log
```

Вывод:

```text
28 logs/2026-02-20.log
28 logs/2026-02-21.log
28 logs/2026-02-22.log
84 total
```

`-l` означает lines, то есть количество строк.

Набери `wc -l logs/*.log`.

Должны быть видны три лога по `28` строк и общая сумма `84`.

## Считать байты: `-c`

Если нужно быстро понять размер файла, удобен флаг `-c`.

```bash
wc -c page.html configs/server.ini
```

Вывод:

```text
539 page.html
173 configs/server.ini
712 total
```

`-c` считает байты.

Набери `wc -c page.html configs/server.ini`.

Должны быть видны размеры `539` и `173`.

## Считать слова: `-w`

Иногда нужен именно грубый подсчёт слов.

```bash
wc -w configs/server.ini page.html
```

Вывод:

```text
13 configs/server.ini
39 page.html
52 total
```

`-w` означает words.

Набери `wc -w configs/server.ini page.html`.

Должны быть видны значения `13` и `39`.

## `wc` очень часто живёт в pipe

Одна из самых полезных ролей `wc` — быть концом цепочки. Сначала другая команда что-то выводит, а `wc` это считает.

Например:

```bash
cat chat/*.log | wc -l
```

Вывод:

```text
10
```

Здесь `cat` собрал строки из двух файлов, а `wc -l` посчитал их общее количество.

Набери `cat chat/*.log | wc -l`.

Должно появиться число `10`.

`wc` часто используют именно так: в конце pipe, когда нужно быстро получить одно число.

## Комбинация с `find`

После прошлого урока уже можно сочетать `find` и `wc`.

```bash
find players/europe -name '*.txt' | wc -l
```

Вывод:

```text
3
```

Здесь `wc -l` считает не строки внутри файлов, а строки вывода `find`. То есть по сути количество найденных путей.

Набери `find players/europe -name '*.txt' | wc -l`.

Должно появиться число `3`.

## Шпаргалка

- `wc файл` показать сразу несколько счётчиков.
- `wc -l файлы` посчитать строки.
- `wc -c файлы` посчитать байты.
- `wc -w файлы` посчитать слова.
- `... | wc -l` посчитать количество строк в выводе другой команды.

## Напоследок

1. GNU `wc` хорошо оптимизирована под большие файлы.
Её не делали “игрушечной”: такие команды часто гоняют по огромным логам, поэтому реализация в coreutils давно заточена под быстрый подсчёт.

2. `wc -l` часто используют как очень грубую метрику размера кода.
Например, чтобы быстро понять, насколько большой проект или файл, даже если такая оценка довольно грубая.

Набери `find . -name '*.csv' | wc -l`.

Должно появиться число `7`.

## Практика

3.1. Посчитай строки в каждом файле из `logs`.

   Должны быть видны:
   - `28 logs/2026-02-20.log`
   - `28 logs/2026-02-21.log`
   - `28 logs/2026-02-22.log`
   - `84 total`

3.2. Посчитай байты у `page.html` и `configs/server.ini`.

   Должны быть видны:
   - `539 page.html`
   - `173 configs/server.ini`
   - `712 total`

3.3. Через `tree` и `wc` посчитай, сколько строк будет в дереве `players`, если вывести его только до файлов игроков.

   Должно появиться число `13`.

3.4. Через `find`, условие `-o` и `wc` посчитай, сколько файлов либо лежат в `players/europe`, либо заканчиваются на `.csv`.

   Должно появиться число `10`.
