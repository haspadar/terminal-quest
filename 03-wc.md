# 03. `wc`: считаем строки, слова и байты

После `find` хочется оценить масштаб: сколько строк в логах, сколько весят файлы, сколько файлов нашлось.

Для этого и нужен `wc`.

Перейдите в папку:

```bash
cd game_server_intro
```

## Название

`wc` расшифровывается как `word count`, то есть “подсчёт слов”. Но на практике команду используют не только для слов: чаще всего для строк, байтов и быстрых подсчётов через pipe.

## Синтаксис

Самый простой вид такой:

```bash
wc файл
```

Например:

```bash
wc logs/2026-02-20.log
```

Вывод:

```text
28 183 1952 logs/2026-02-20.log
```

Без флагов `wc` показывает сразу несколько чисел. Обычно это:
- строки;
- слова;
- байты.

Если нужен один конкретный счётчик, лучше взять флаг.

## Считать строки: `-l`

Чаще всего `wc` используют именно для строк.

```bash
wc -l logs/*.log
```

Вывод:

```text
28 logs/2026-02-20.log
28 logs/2026-02-21.log
28 logs/2026-02-22.log
84 total
```

`-l` означает lines, то есть количество строк.

## Считать байты: `-c`

Если нужно быстро понять размер файла, удобен флаг `-c`.

```bash
wc -c page.html configs/server.ini
```

Вывод:

```text
539 page.html
173 configs/server.ini
712 total
```

`-c` считает байты.

## Считать слова: `-w`

Иногда нужен грубый подсчёт слов.

```bash
wc -w configs/server.ini page.html
```

Вывод:

```text
13 configs/server.ini
39 page.html
52 total
```

`-w` означает words — слова.

## `wc` очень часто живёт в pipe

`wc` часто стоит в конце цепочки: другая команда выводит, а `wc` считает.

Например:

```bash
cat chat/*.log | wc -l
```

Вывод:

```text
10
```

`cat` собрал строки из двух файлов, `wc -l` посчитал общее количество.

`wc` часто используют именно так: в конце pipe, когда нужно быстро получить одно число.

## Комбинация с `find`

`find` и `wc` хорошо работают вместе.

```bash
find players/europe -name '*.txt' | wc -l
```

Вывод:

```text
3
```

`wc -l` считает строки вывода `find` — то есть количество найденных путей.

## Шпаргалка

- `wc файл` показать сразу несколько счётчиков.
- `wc -l файлы` посчитать строки.
- `wc -c файлы` посчитать байты.
- `wc -w файлы` посчитать слова.
- `... | wc -l` посчитать количество строк в выводе другой команды.

## Напоследок

1. GNU `wc` оптимизирована под большие файлы: реализацию в coreutils давно затачивали под быстрый подсчёт в огромных логах.

2. `wc -l` часто используют как грубую метрику размера кода — чтобы быстро оценить масштаб проекта или файла.

## Практика

3.1. Посчитайте строки в каждом файле из `logs`.

   Должны быть видны:
   - `28 logs/2026-02-20.log`
   - `28 logs/2026-02-21.log`
   - `28 logs/2026-02-22.log`
   - `84 total`

3.2. Посчитайте байты у `page.html` и `configs/server.ini`.

   Должны быть видны:
   - `539 page.html`
   - `173 configs/server.ini`
   - `712 total`

3.3. Через `tree` и `wc` посчитайте, сколько строк будет в дереве `players`, если вывести его только до файлов игроков.

   Должно появиться число `13`.

3.4. Через `find`, условие `-o` и `wc` посчитайте, сколько файлов либо лежат в `players/europe`, либо заканчиваются на `.csv`.

   Должно появиться число `10`.
