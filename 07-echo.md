# 07. `echo`: быстро показываем значения

Переменную вы уже умеете сохранить. Следующая задача проще: быстро посмотреть, что именно в ней лежит. Для этого нужен `echo`.

`echo` печатает свои аргументы на экран. Его удобно использовать, когда надо:

- проверить переменную;
- быстро собрать строку из текста и значения;
- посмотреть, какой список файлов вернула команда.

Перейдите в папку:

```bash
cd game_server_intro
```

Дальше все команды запускайте внутри неё.

## Базовый вывод

У `echo` простая форма:

```bash
echo text
```

Например:

```bash
echo players/europe
```

Должен появиться путь `players/europe`.

`echo` просто выводит аргументы, которые вы ему передали.

Наберите:

```bash
echo scan: players/europe
```

Должна появиться строка `scan: players/europe`.

## Показать переменную

После `set` проверить значение проще всего через `echo`.

```bash
set folder players/europe
echo $folder
```

Должен появиться путь `players/europe`.

Так удобно быстро убедиться, что в переменной лежит нужное значение.

```bash
set report chat/global_chat.log
echo file: $report
```

Должна появиться строка `file: chat/global_chat.log`.

## Показать список значений

В `fish` переменная может хранить список. `echo` покажет все элементы сразу.

```bash
set files (find players/europe -name '*.txt' | sort)
echo $files
```

Должны быть видны:

- `players/europe/anna.txt`
- `players/europe/ivan.txt`
- `players/europe/max.txt`

Здесь список выводится в одну строку, через пробелы.

Наберите:

```bash
set logs (find logs -name '*.log' | sort)
echo $logs
```

Должны быть видны:

- `logs/2026-02-20.log`
- `logs/2026-02-21.log`
- `logs/2026-02-22.log`

## Без перевода строки: `-n`

Обычно `echo` добавляет перевод строки в конце. Иногда это не нужно.

```bash
echo -n scan:
echo players/europe
```

Должна появиться строка `scan:players/europe`.

Флаг `-n` убирает завершающий перевод строки.

Наберите:

```bash
echo -n log:
echo chat/global_chat.log
```

Должна появиться строка `log:chat/global_chat.log`.

## Спецсимволы: `-e`

Если нужно явно вставить перевод строки или табуляцию прямо в строку, пригодится `-e`.

```bash
echo -e "anna\nivan\nmax"
```

Должны быть видны:

- `anna`
- `ivan`
- `max`

Здесь `\n` значит перевод строки.

Наберите:

```bash
echo -e "region:\tplayers/europe"
```

Должна появиться строка, где после `region:` будет табуляция и `players/europe`.

## Быстро дать данные в pipe

`echo` часто используют как самый простой источник тестовых данных для пайпа.

```bash
echo error | wc -c
```

Должно появиться число `6`.

Здесь в `wc -c` уходит строка `error` и завершающий перевод строки.

Наберите:

```bash
echo error | grep err
```

Должна вывестись строка `error`.

Так удобно быстро проверить, как работает следующая команда в пайпе, не создавая отдельный файл.

## Быстро записать строку в файл

`echo` часто используют в скриптах и CI, когда нужно быстро создать маленький файл или дописать в него строку.

```bash
echo hello > /tmp/echo-demo.txt
cat /tmp/echo-demo.txt
```

Появится файл с текстом `hello`.

Здесь `>` перенаправляет вывод в файл.

## Почему `echo *` выводит список файлов

Иногда `echo` используют так:

```bash
echo *
```

Если в папке много файлов, команда выведет их все в одну строку.

Это не специальная функция `echo`. Сначала shell подставляет вместо `*` имена файлов, и только потом передаёт этот список в `echo`.

## Когда `echo` уже не хватает

`echo` удобен для быстрой проверки, но у него простая задача: вывести текст как можно быстрее.

У `echo` нет одного полностью надёжного поведения для всех оболочек и систем. Поведение зависит от системы: флаги вроде `-n` и `-e`, а также строки со спецсимволами могут работать по-разному.

Например:

```bash
echo "\nhello"
```

В одной системе это может вывести буквально `\nhello`, а в другой — сделать перенос строки перед `hello`.

Поэтому, когда нужен аккуратный и предсказуемый шаблон вывода, лучше брать `printf`. Следующий урок как раз про это.

## Шпаргалка

- `echo text` вывести текст.
- `echo $name` вывести значение переменной.
- `echo text $name` вывести текст и значение в одной строке.
- `echo -n text` вывести текст без перевода строки.
- `echo -e "a\nb"` обработать спецсимволы внутри строки.

## Напоследок

1. `echo hello` — это буквально одна из первых команд, которые учат в терминале.

2. Когда нужно быстро проверить пайп, проще всего написать `echo test | wc -c` или `echo error | grep err`. Поэтому `echo` в терминале часто работает как маленький генератор входных данных для других команд.
