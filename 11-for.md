# 11. `for`: повторяем команду по списку

Когда нужно выполнить одну и ту же команду для нескольких файлов или значений — пишут цикл `for`. Без него пришлось бы повторять команду вручную для каждого элемента.

Перейдите в папку:

```bash
cd game_server_intro
```

## Базовый синтаксис

В fish цикл выглядит так:

```bash
for x in item1 item2 item3
    echo $x
end
```

Переменная `x` принимает значения из списка по очереди. После `end` цикл завершается.

Попробуйте:

```bash
for region in europe asia america
    echo $region
end
```

Должны появиться три строки: `europe`, `asia`, `america`.

## Цикл по файлам

Shell подставляет файлы по маске сам — до запуска цикла. `find` и `ls` не нужны:

```bash
for f in logs/*.log
    echo $f
end
```

Должны появиться:

```
logs/2026-02-20.log
logs/2026-02-21.log
logs/2026-02-22.log
```

Не пишите `for f in (ls *.log)` — лишний процесс, ломается на именах с пробелами. Просто `for f in *.log`.

## Команда внутри цикла

Внутри тела цикла можно запускать любую команду с переменной:

```bash
for f in logs/*.log
    wc -l $f
end
```

Должно появиться количество строк для каждого файла:

```
28 logs/2026-02-20.log
28 logs/2026-02-21.log
28 logs/2026-02-22.log
```

## Однострочная форма

Цикл можно писать в одну строку — удобно в терминале:

```bash
for f in logs/*.log; echo $f; end
```

Должны появиться те же три пути. Результат идентичен многострочному варианту.

## Список из команды

Результат команды можно передать в `for` через подстановку:

```bash
for f in (find players -name "*.txt" | sort)
    echo $f
end
```

Должны появиться все семь файлов игроков в алфавитном порядке.

Попробуйте заменить `echo $f` на `wc -c $f` — увидите размер каждого файла в байтах.

## Префикс в выводе

Цикл удобен для форматирования. Например, добавить метку к каждому пути:

```bash
for f in (find players -name "*.txt" | sort)
    echo "profile: $f"
end
```

Должна появиться строка `profile:` перед каждым путём.

## Числовые диапазоны

В bash есть `{1..5}` и `for (( i=1; i<=5; i++ ))`. В fish это не работает. Для числового диапазона используют `seq`:

```bash
for i in (seq 1 5)
    echo $i
end
```

Должны появиться числа от 1 до 5.

`seq` принимает три аргумента: `seq start step end`. Например, каждое второе число:

```bash
for i in (seq 1 2 10)
    echo $i
end
```

Должны появиться: `1 3 5 7 9`.

## Шпаргалка

- `for x in list ... end` — пройти по всем значениям.
- `for f in *.log` — перебрать файлы по маске.
- `for f in (команда)` — перебрать результат команды.
- `for i in (seq 1 5)` — числовой диапазон.
- Внутри тела цикла переменная доступна как `$x` или `$f`.

## Напоследок

- `for` в shell появился раньше, чем в большинстве языков программирования — ещё в Bourne shell в 1970-х. Синтаксис намеренно минимальный: большая часть работы перекладывается на расширения shell — glob и подстановки — которые выполняются до запуска цикла.
- Иногда `for` вообще не нужен: `rm *.log` короче цикла с `rm "$f"`. Цикл оправдан, когда внутри тела нужно сделать что-то нетривиальное.

## Практика

11.1. Сохраните в переменную отсортированный список `.txt` файлов из `players/europe` и выведите каждый путь в цикле.

   Должны появиться три строки: пути к `anna.txt`, `ivan.txt`, `max.txt`.

11.2. Сохраните в переменную список `.log` файлов и в цикле посчитайте строки в каждом через `wc -l`.

   Должны появиться три строки с количеством строк и именем файла.

11.3. В цикле пройдите по всем `.txt` файлам игроков и выведите каждый путь с префиксом `profile:`.

   Должны появиться семь строк вида `profile: players/...`.

11.4. В цикле пройдите по всем `.log` файлам и выведите только те, в которых встречается слово `AuthFailed` (используйте `grep -l`).

   Должны появиться имена файлов, содержащих `AuthFailed`.
